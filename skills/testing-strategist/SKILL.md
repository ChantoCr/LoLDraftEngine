# testing-strategist

## Purpose
Define a strong testing strategy for the platform.

## Responsibilities
- identify units that must be deterministic and covered
- propose test pyramids for domain logic, API contracts, and UI flows
- define scenario-based tests for draft states and recommendation outputs
- ensure confidence in patch-aware and mode-aware behaviors

## Rules
- prioritize tests for pure scoring and analysis logic
- treat recommendation regressions as high value test targets
- use realistic fixture drafts
